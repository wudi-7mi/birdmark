# Birdmark 开发任务清单

1. **整理现有 AI 服务**（已完成）  
   把现在的检测、识别接口整理成独立的 AI 推理服务。

2. **新建业务后端**（已完成基础骨架）  
   建一个新的业务 API 服务，专门处理用户、图片、图鉴和任务。

3. **打通两个后端**（已完成基础调用）  
   业务后端通过内部接口调用 AI 推理服务，不直接加载模型。

4. **加入数据库**（已完成基础表）  
   先建用户、图片、观察记录、鉴定结果、物种、图鉴这些基础表。

5. **实现图片存储**（已完成基础存储）  
   保存原图、裁剪图和缩略图，数据库只记录路径。

6. **实现单图上传识别**（已完成基础链路）  
   用户上传一张图，系统保存图片、调用 AI、保存结果。

7. **实现图片详情**（已完成基础详情）  
   可以查看原图、检测框、裁剪图和 AI 鉴定建议。

8. **实现用户确认**（已完成基础接口）  
   用户选择正确物种，或者手动修正、标记未知、标记误检。

9. **生成我的图鉴**（已完成基础聚合）  
   用户确认后，把物种加入自己的鸟类图鉴。

10. **实现登录注册**（已完成基础接口）
    所有内容都需要登录后访问。

11. **实现共享相册**（已完成基础接口）
    登录用户可以浏览其他用户上传的图片和鉴定结果。

12. **实现我的上传**（已完成基础列表和删除）
    用户可以查看和管理自己上传过的图片。

13. **实现批量导入**（已完成基础任务接口）
    支持一次上传多张图片，后台逐张识别并显示进度。

14. **设计 iOS App 第一版**（已完成 V1 文档）
    先做登录、选图、上传、查看结果、确认鉴定、我的图鉴。

15. **完善 iOS 体验**  
    再做共享相册、批量上传、拍照上传、缓存和通知。
